# AndroidUSBCamera 有效修改汇总

> 整理自 MTK 多路摄像头预览问题修复、构建升级及 HP/SP 双摄像头兼容性调优全过程。  
> 参考：[UVCAndroid PR #145](https://github.com/shiyinghan/UVCAndroid/pull/145)

---

## 一、问题背景

| 现象 | 说明 |
|------|------|
| 部分 MTK 手机第二路 UVC 预览黑屏 | 多路摄像头时第二路 `dwMaxPayloadTransferSize` / bulk 传输异常 |
| HP 摄像头可预览、SP 不行（或反之） | 全局启用 MTK fix 会「救活」一种、破坏另一种 |
| Android Studio 无法同步 | 原 AGP 4.2.2 低于 IDE 最低要求 7.1.0 |
| `ndk-build` 路径为 null | `local.properties` 未配置 NDK |
| AAR 集成后 `loadLibrary` 崩溃 | `libuvc-release.aar` 未打包 `.so` |
| Release 编译失败 | Kotlin 空安全、缺少 import 等 |

---

## 二、核心功能修改（UVC / Native）

### 2.1 `libuvc/src/main/jni/libuvc/include/libuvc/libuvc_internal.h`

- 新增 `UVC_QUIRK_FIX_BANDWIDTH`（`0x00000080`）、`UVC_FMT_FLAG_*` 常量。
- 在 `uvc_device_handle` 结构体中增加 `uint32_t quirks` 字段。
- `LIBUVC_NUM_TRANSFER_BUFS` 保持 **10**（与原工程一致；未采用 PR 中 Android 16 的方案，避免影响已有设备）。

### 2.2 `libuvc/src/main/jni/libuvc/src/stream.c`

新增 `uvc_fixup_video_ctrl()`，在 `uvc_query_stream_ctrl()` 的 GET 路径末尾调用，用于修正流协商参数：

1. **Elgato Cam Link 4K** 错误 probe 响应修正（特定 VID/PID）。
2. **`dwMaxVideoFrameSize` 为 0** 时从 frame 描述符补全。
3. **`dwMaxPayloadTransferSize` 符号扩展错误** 修正（高 16 位全 1 时清零）。
4. **等时（isochronous）非压缩流 + 已启用 quirks**：按分辨率/帧率重算带宽（MTK 多路关键修复）。
5. **bulk 传输 + 已启用 quirks**：将 `dwMaxPayloadTransferSize` 限制为 **16384**（Android usbfs  workaround）。

**重要**：带宽重算与 bulk 限制均仅在 `devh->quirks & UVC_QUIRK_FIX_BANDWIDTH` 时生效，**默认不改动**原有摄像头行为。

### 2.3 `libuvc/src/main/java/com/jiangdg/uvc/UVCCamera.java`

- 新增 `UVC_QUIRK_FIX_BANDWIDTH` 常量。
- 新增 `getRecommendedPlatformQuirks()`：MTK 平台（`Build.HARDWARE` 以 `mt` 开头或含 `mediatek`）返回该 quirk，否则返回 0。
- `open(UsbControlBlock)` 默认 **quirks = 0**（兼容 HP 等原有摄像头）。
- 新增 `open(UsbControlBlock, int quirks)`，通过 JNI 传入 native 层。
- `nativeConnect` 签名增加 `int quirks` 参数。

### 2.4 JNI / C++ 层

| 文件 | 修改 |
|------|------|
| `libuvc/.../UVCCamera.h` | `connect(..., int quirks)` |
| `libuvc/.../UVCCamera.cpp` | 连接成功后 `mDeviceHandle->quirks = quirks` |
| `libuvc/.../serenegiant_usb_UVCCamera.cpp` | `nativeConnect` 增加 `jint quirks`，JNI 签名 `(JIIIIILjava/lang/String;I)I` |

## 三、上层兼容策略（HP + SP 双摄像头）

### 3.1 设计原则

- **第一路（slot 0）**：默认 `quirks = 0`，保证 HP 类摄像头正常。
- **第二路及以后（slot ≥ 1）**：在 MTK 上自动启用 `getRecommendedPlatformQuirks()`，修复 SP 类第二路预览。
- **单独使用 SP 且失败**：自动以 platform quirks **重试一次**。
- **手动覆盖**：通过 `CameraRequest.setUvcQuirks()` 指定。

### 3.2 `libausbc/.../camera/bean/CameraRequest.kt`

- 新增字段 `uvcQuirks: Int? = null`（null 表示走自动策略）。
- 新增 `Builder.setUvcQuirks(quirks: Int)`。

### 3.3 `libausbc/.../MultiCameraClient.kt`

- `openCamera()` 时调用 `allocateUvcOpenSlot()` 分配并发打开序号。
- `closeCameraInternal()` 路径中 `CameraUVC` 调用 `releaseUvcOpenSlot()` 释放序号。
- `resolveUvcQuirks(request, openSlot)`：
  - 若 `request.uvcQuirks != null` → 使用指定值；
  - 若 `openSlot > 0` → 返回 `getRecommendedPlatformQuirks()`；
  - 否则 → 0。
- `ICamera` 增加 `mUvcOpenSlot` 字段。

### 3.4 `libausbc/.../camera/CameraUVC.kt`

- 重构为 `tryOpenAndPreview()` + 失败重试逻辑。
- 打开时：`resolveUvcQuirks(request, mUvcOpenSlot)`。
- 若失败且未手动指定 quirks：用 `getRecommendedPlatformQuirks()` 再试一次。
- 关闭时释放 open slot。

### 3.5 `libausbc/.../camera/CameraUvcStrategy.kt`

- 补充 `import com.jiangdg.ausbc.MultiCameraClient`。
- `open()` 使用 `MultiCameraClient.resolveUvcQuirks(request, 0)`。

### 3.6 手动配置示例（可选）

```kotlin
// 仅当某路摄像头（如 SP）单独使用仍黑屏时
CameraRequest.Builder()
    .setPreviewWidth(640)
    .setPreviewHeight(480)
    .setUvcQuirks(UVCCamera.UVC_QUIRK_FIX_BANDWIDTH)
    .create()
```

## 五、NDK 与 AAR 打包修复

### 5.1 `libuvc/build.gradle`

- **`getNdkDir()`**：按优先级解析 NDK 路径  
  `NDK_HOME` → `local.properties ndk.dir` → `sdk.dir/ndk/<version>` → 已安装最高版本。
- **`ndkVersion`**：与根工程 `ext.ndkVersion` 对齐（Apple Silicon 需 NDK 23+，推荐 26）。
- **`sourceSets.main.jniLibs.srcDir 'src/main/libs'`**：使用 ndk-build 输出目录。
- **构建依赖**：`preBuild`、`merge*JniLibFolders`、`bundle*Aar` 均依赖 `ndkBuild`，确保 AAR 内含 `.so`。
- `lintOptions` → `lint { }`（AGP 7 语法）。

### 5.2 AAR 集成要点

对外分发需 **同时提供** 以下 Release AAR（`libausbc` 不会合并依赖）：

```
libausbc/build/outputs/aar/libausbc-release.aar      # 上层 API
libuvc/build/outputs/aar/libuvc-release.aar        # 含 jni/*.so，必须
libuvccommon/build/outputs/aar/libuvccommon-release.aar
```

构建命令：

```bash
bash gradlew \
  :libuvc:clean :libuvc:assembleRelease \
  :libuvccommon:assembleRelease \
  :libausbc:assembleRelease
```

验证 `libuvc` AAR 是否含 native 库（约 650KB，含 `jni/arm64-v8a/*.so`）：

```bash
unzip -l libuvc/build/outputs/aar/libuvc-release.aar | grep "\.so"
```

## 八、验证结果

- MTK 平台 HP + SP **双路预览均可工作**（先开 HP / 同时开，第二路自动 quirks）。
- Release AAR 可正常构建，`libuvc` AAR 含完整 native 库。
- Android Studio 可正常 Sync（AGP 7.4.2）。

## 九、未完全解决 / 注意事项

1. **`libnative` 模块**：在 NDK 26 下 LAME 源码编译仍有 C99 告警/错误；若需 MP3 功能可暂用 `libnative/aar/libnative-3.2.9.aar` 预编译包，或后续单独修复 CMake 编译。
2. **修改 native 代码后**必须重新执行 `:libuvc:assembleRelease`，并替换目标项目中的 `libuvc-release.aar`。
3. **目标项目** `local.properties` 需配置 `sdk.dir`；源码集成方式需本机安装 NDK 26+（Mac Apple Silicon）。
4. 若某摄像头**单独使用**仍黑屏且自动重试无效，请对该路显式调用 `setUvcQuirks(UVCCamera.UVC_QUIRK_FIX_BANDWIDTH)`。

---

## 十、给其他项目集成的最简步骤

1. 构建并拷贝 3 个 Release AAR（见第五节）。
2. 目标 `app/build.gradle` 使用 `flatDir` 引入三个 AAR 及 xlog 等依赖。
3. `AndroidManifest` 声明 `usb.host` 与 `CAMERA` 权限。
4. 业务代码继承 `CameraFragment` / `MultiCameraFragment` 或直接使用 `CameraUVC`。
5. 多路场景默认无需额外配置；特殊摄像头按需 `setUvcQuirks`。
